教育用の概念モデルです。

送信側 flow 消費の模式図（レイテンシーA/B共通）

左の送信ノードから、右の各受信先へ 1受信先あたり ceil(購読ch / 4) 本の unicast flow が必要、という見せ方です。 受信先が増えると、その分だけ送信側の flow 消費も増えます。

読み取りのポイント

  • 1ch購読でも1flowなので、帯域は「1ch分」ではなく4ch枠で確保されると考える。
  • 1～4chまでは同じ1flow。5chになった瞬間に2flowへ増える。
  • 受信先が増えると、ユニキャストでは送信側のflow数も増えるので、送信側のppsと総帯域が増える。
  • レイテンシーを下げると、各flowの packet 化が細かくなるため、同じflow数でも pps が増える。

新人向けの言い換え

  • Danteユニキャストの帯域は「人数」ではなく4席単位で会議室を借りる感じ
  • 1人でも4席室を1部屋借りる
  • 5人なら4席室を2部屋借りる
  • 受信先が増えるのは、その会議室を別の建物ごとに追加で借りるイメージ
※ オーバーヘッドは簡略化して Ethernet + FCS + Preamble/SFD + IFG + IPv4 + UDP + RTP ≒ 78 byte / packet として扱っています。PTP、制御通信、VLAN、冗長、実装差などは省略しています。
